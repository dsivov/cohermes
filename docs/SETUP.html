Before you start

Prerequisites

  • A running Context Graph server (the backend), e.g. http://10.0.0.80:9621.
  • Claude Code installed and logged in on a Pro / Max / Team subscription — not an API key. Check: claude auth status should show "authMethod": "claude.ai".
  • Python 3.12. For the Hermes-orchestrated flow, also the Hermes prerequisites (a model provider).
The mental model
One project = one shared Context Graph workspace (the team brain). Every developer runs their own Claude Code agent locally; all agents read from and write to that one workspace. cohermes is the glue — a small cg/ package + a local MCP server + a Hermes hook.
1install

Install cohermes (each developer)

# clone the fork and install — this brings Hermes AND the cg extension
git clone https://github.com/dsivov/cohermes && cd cohermes
python -m venv .venv && . .venv/bin/activate
pip install -e .

This gives the commands: cohermes-agent, cohermes-mcp, cohermes-preset, cohermes-backfill, cohermes-ontology, cohermes-install-hook — plus the hermes CLI. The cg package imports with no PYTHONPATH once installed.

2connect

Connect to the shared brain

# point at your CG server + the project's workspace
export COHERMES_CG_URL=http://<cg-host>:9621
export COHERMES_CG_WORKSPACE=<your-project>

cohermes-ontology                          # install the shared ontology (once, any dev)
cohermes-backfill /path/to/project --docs README.md   # optional: seed the brain (curated)

The ontology defines the shared shape (Decision · Task · Commit · Review · Insight · Module + their links). Backfill is curated — it ingests module purposes + design docs, not raw code.

3the key decision

Integration mode: claude -p vs tmux

There are two separate sessions: Hermes's own session (state in ~/.hermes) and Claude Code's session (its own conversation/context, which Claude Code stores and can resume). How context reaches Claude depends on the mode.

-p (print)tmux (interactive)
Claude contextfresh per call (unless --resume)persists in the live session
How context arrivesyou inject it (prompt · --append-system-prompt · --resume)Claude keeps it; Hermes relays new turns
Best fortask-scoped automation, CI, one-shota developer's live multi-turn session
Brief injectionclean, at launch (cohermes-agent)at session start; re-orient mid-session
Orchestrationsimpleheavier (send-keys / dialog handling)
Recommendation — -p as primary
Use -p with the brief injected at launch (that's cohermes-agent): one call per task, the agent starts aligned, and cross-task continuity comes from the brain, not the session. Reach for tmux when a developer wants a live, multi-turn Claude session — inject the brief at start and keep orient available as an MCP tool. Either way, durable knowledge lives in Context Graph, so you never depend on Hermes-session-history reaching Claude.
4per developer

Wire each developer's project (distinct identity)

# Alice
COHERMES_DEVELOPER=alice cohermes-preset /path/to/repo-A --developer alice
# Bob
COHERMES_DEVELOPER=bob   cohermes-preset /path/to/repo-B --developer bob

Each repo gets a .mcp.json (the context-graph + local cohermes MCP servers), a CLAUDE.md loop stamped with that developer, the /cg-* slash-commands, and MCP pre-approval. Writes are attributed to that developer (D2).

5run

Run an agent

Pre-briefed (recommended)

cd /path/to/repo-A
cohermes-agent "add a gitlab review source"          # interactive, pre-briefed from the brain
cohermes-agent -p "how should we test a connector?"  # one-shot

cohermes-agent runs the subscription-auth preflight, fetches the orient brief, and launches Claude Code with it injected — so the agent begins with the team's decisions + learnings automatically.

Plain Claude Code (the tools are still there)

cd /path/to/repo-A && claude
# then, in-session:
/cg-orient "webhook connector auth"   # brief yourself from the brain
# the cohermes MCP tools: record_decision, record_task/commit/review, link,
# ingest_context, record_learning, orient
6hermes runtime

The Hermes hook (optional, for Hermes-orchestrated flow)

cohermes-install-hook     # → ~/.hermes/hooks/cohermes/ (auth preflight + orient @ agent:start)
hermes                    # run Hermes; the hook fires on each task

On agent:start the hook asserts subscription auth and writes the orient brief to $HERMES_HOME/cohermes/brief-<session>.md. (Hermes hooks are side-effect only; for automatic prompt-injection, prefer cohermes-agent, which injects the brief at launch.)

7the payoff

Two-developer walkthrough

Alice (repo-A): cohermes-agent "choose webhook auth" — her agent decides OAuth 2.0 and records it (record_decision, stamped alice).

Bob (repo-B, later): cohermes-agent "implement webhook token refresh" — his agent is pre-briefed, sees Alice's OAuth decision, and builds on it (cites her, doesn't re-derive), recording its own follow-on stamped bob.

Success
Two developers' work lands in one attributed graph; Bob's agent reuses Alice's decision; everything runs on subscription auth; the chain is traceable and learnings retrievable.
8see it

The UI

Open the Context Graph WebUI: http://<cg-host>:9621/webui (workspace <your-project>). Watch both developers' decisions, the Decision→Task→Commit→Review chain, and Insight learnings appear in one graph. If you orchestrate through Hermes, its TUI / web dashboard is the agent UI.

if something's off

Troubleshooting

SymptomFix
Agent uses metered API billingunset ANTHROPIC_API_KEY (and cloud-provider vars); claude auth status must show claude.ai. cohermes-agent hard-stops on this.
MCP server "pending approval"the preset writes .claude/settings.local.json with enableAllProjectMcpServers; run claude once to approve if needed.
cg import error / MCP won't startensure you pip install -e . (not just a checkout) — that removes the PYTHONPATH requirement.
Agent says "no cohermes tools are exposed" / falls back to context-graph's record_decision (trace, no node)the cohermes stdio MCP server failed to start — almost always a missing mcp package. Confirm with claude mcp list (both context-graph and cohermes must appear) and python -m cg.mcp_server. Do a full pip install -e . (not --no-deps) so mcp + requests are installed.
Empty orient briefseed the brain first — cohermes-ontology + cohermes-backfill, and record a decision or two.
